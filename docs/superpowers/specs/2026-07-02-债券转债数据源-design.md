# 债券与可转债数据源设计

- **日期**: 2026-07-02
- **主题**: 为 local-datasource MCP server 新增「中国境内债券 + 可转债」数据源
- **状态**: 待实现
- **底层数据源**: akshare(项目已依赖,免费、无需 apikey)

## 背景与目标

local-datasource 是一个 stdio 模式的 Python MCP server,已有 A股/HK股/美股(yfinance)/World Bank/arXiv 共 4 个 tool。现需新增债券与可转债领域数据接口,服务于信评与转债分析场景。

核心需求:
- 标债(利率债 + 信用债):国债收益率曲线、信用债发行信息、信用债交易所行情
- 可转债:实时一览含溢价率、历史K线、条款信息(强赎/回售/下修/剩余期限)、发行人财务数据
- 约束:免费、尽量不配置 apikey、用量充足

经调研(GitHub + akshare 官方文档),akshare 是唯一同时覆盖利率债/信用债/可转债的免费库(20.9k★,月度迭代,无 key)。本项目 A股 provider 已在用 akshare,故新增债券/转债为无缝衔接,零新增依赖。

## 设计决策

### 为什么用 akshare

| 库 | 需 key | 债券覆盖 | 转债覆盖 | 活跃度 | 结论 |
|---|---|---|---|---|---|
| akshare (20.9k★) | 否 | 国债收益率曲线/企业债/城投债/国债期货 | 全覆盖(一览/K线/溢价率/条款/集思录) | 月度迭代 | **首选** |
| efinance (3.8k★) | 否 | 东财债券 | 覆盖 | 活跃 | 次选,本期不用 |
| Tushare Pro | **必须 token** | 债券接口高积分档收费 | 高积分档收费 | GitHub 停更2年 | ❌ 不免费 |
| baostock | 否 | 几乎无 | 不支持 | — | ❌ 不覆盖 |
| pytdx | 否 | — | — | 已归档 | ❌ 废弃 |

### 为什么按「领域分 2 个 tool」而非 1 个或 6 个

- 现有项目模式:一个数据域 = 一个 tool = 一个 provider 文件(worldbank/arxiv 独立,yfinance 独立);`query_stock` 是唯一的「一个 tool + 分流」模式,但它的 `market` 分流是**同构查询**(A/HK/US 参数结构一致)。
- 本期 6 类查询是 **6 件异构的事**(收益率曲线只要日期、转债一览无参数、历史K要转债代码、货币网要查询条件…),硬塞进一个 `query_bond(kind=...)` 会出现条件必填逻辑,schema 不干净。
- 但完全拆成 6 个 tool 又过碎(tool 4→10),且域内查询有一定同构性。
- **折中**:按数据域分 2 个 tool(`query_bond` + `query_convertible_bond`),域内用 `kind` 分流。tool 4→6,贴合现有「一个数据域一个 tool」精神。

### 能力边界:务实分层(实测确认)

akshare 免费层在标债深度数据上有真实缺口,经 context7 抓官方文档 + 实测 `2180495.IB` 逐项确认:

| 能力 | 转债 | 标债 |
|---|---|---|
| 条款/发行信息 | ✓ `bond_zh_cov` + `bond_cb_redeem_jsl` | ⚠️ 部分:`bond_info_cm` 有发行人/类型/发行日/评级;赎回回售条款详情、票息、到期日=免费层拿不到 |
| 剩余期限 | ✓ `bond_cb_summary`(集思录,含剩余期限/到期税前收益) | ✗ akshare 不直接提供到期日,算不出 |
| 估值(交易信息) | ✓ `债现价`(交易所成交价) | ✗ 中债估值 YTM/全价=Wind/中债登收费,akshare 免费层无;只有交易所成交价 |
| 发行人财务 | ✓ 正股是上市公司→`stock_financial_report_sina` | ⚠️ 上市发行人可走正股;城投/非上市发行人财务=免费层无 |

策略:转债做满;标债做 akshare 能取的(发行信息+评级+交易所成交价),缺口在 tool `description` 与返回 summary 中诚实标注为已知限制,不假装能做。

## 架构与组件

### 文件结构

```
src/local_datasource/providers/
├── bond.py              ← 新增:标债域
└── convertible_bond.py  ← 新增:转债域
```

每个 provider 暴露一个函数,统一签名 `query_xxx(...) -> tuple[str, str]`,返回 `(file_path, summary)`,内部 `kind` 路由,最终走现有 `formatters.format_csv_output(df, file_path)` 写 CSV。与 `stock.py` / `arxiv.py` 完全一致。

### tool 1: `query_bond(kind, …, file_path)` — 标债域

| kind | 用途 | akshare 接口 | 关键参数 | 能取到 | 已知限制 |
|---|---|---|---|---|---|
| `yield_curve` | 国债到期收益率曲线 | `bond_china_yield(start_date, end_date)` | `start_date`, `end_date` (`YYYY-MM-DD`) | 各期限到期收益率(1/3/5/7/10年) | 无 |
| `issue_info` | 信用债发行信息 | `bond_info_cm(bond_code=…)` | `bond_code` (支持 `2180495.IB` / `2180495`) | 发行人/类型/发行日/最新评级/查询代码 | `bond_info_detail_cm` 有上游 bug(见下),不调用;赎回回售条款详情/票息/到期日/剩余期限=免费层无 |
| `credit_daily` | 信用债交易所日行情 | `bond_zh_hs_daily(symbol, start_date, end_date)` | `symbol` (如 `sh019623`), `start_date`, `end_date` | 开高低收量(交易所成交价) | 中债估值 YTM/全价=免费层无 |

### tool 2: `query_convertible_bond(kind, …, file_path)` — 转债域

| kind | 用途 | akshare 接口 | 关键参数 | 字段 |
|---|---|---|---|---|
| `overview` | 一览+溢价率 | `bond_zh_cov()` | 无(可选 `keyword` 过滤) | 代码/正股/转股价/正股价/转股价值/债现价/转股溢价率/规模/评级 |
| `terms` | 条款+剩余期限 | `bond_cb_redeem_jsl()` + `bond_cb_summary()` | 无(全市场一览) | 强赎/回售/下修状态 + 剩余期限 + 到期税前收益 |
| `history` | 历史 K 线 | `bond_zh_hs_cov_daily`(`period=daily`) / `bond_zh_hs_cov_min`(`period=min`) | `symbol` (如 `sz128039`), `start_date`, `end_date`, `period` | 开高低收量 |
| `issuer_finance` | 发行人财务 | `bond_zh_cov()` 解析正股 → `stock_financial_report_sina(stock, symbol)` | `bond_code`/`stock_code` 二选一 + `report_type` | 资产负债表/利润表/现金流量表 |

### `issuer_finance` 的 bond_code / stock_code 二选一

- `bond_code`(转债)→ 内部调 `bond_zh_cov()` 全表过滤出对应「正股代码」→ 再调 `stock_financial_report_sina` 取三大报表
- `stock_code`(任意上市发行人,如央企债发行人)→ 直接调 `stock_financial_report_sina`
- 城投/非上市发行人(实测 `2180495` = 徐州新盛,城投平台,未在 A股上市)→ 解析正股失败,**不报错**,返回引导性提示(见错误处理)

## 关键实现细节(实测固化)

### 1. ticker 归一化

新增 `_normalize_bond_code(code)` 与 `_normalize_cb_code(code)`,沿用 `stock.py:27-46` 的 `_normalize_a_code` 模式:
- 标债 `bond_code`:剥掉 `.IB`/`.SH`/`.SZ` 后缀,只传数字部分给 `bond_info_cm(bond_code=…)`(akshare 不认带后缀的代码)
- 转债 `symbol`:接受 `sz128039` / `128039` / `113682.SH` 等多种写法,统一归一到 akshare 原生格式

### 2. issue_info 的精确过滤(防子串误匹配)

实测发现 `bond_info_cm(bond_code="2180495")` 返回 2 条:
- `21徐州新盛03` / 代码 `2180495` (目标)
- `21赣州银行CD111` / 代码 `112180495` (误命中,子串匹配)

provider 必须在返回前做精确过滤:`df = df[df['债券代码'] == normalized_code]`,过滤后为空则 `raise ValueError`。

### 3. 放弃 `bond_info_detail_cm`

该接口存在 akshare 上游 bug(内部 `bond_info_cm(bond_name=…)` 调用列名与中国货币网返回对不上,直接 KeyError)。本期不调用,只用 `bond_info_cm` 主查询。spec 中标注为已知上游缺陷,后续 akshare 修复后可作为可选增强。

### 4. 城投/非上市发行人财务缺口处理

`issuer_finance` 解析正股失败时(如 `2180495` 城投平台),不报错,返回单行提示 CSV:
> "发行人 {name} 为非上市主体,免费层无财务数据,建议查 Wind/企业预警通"

符合「反馈引导行动」原则,而非静默返回空。

## 错误处理

沿用项目现有风格(`if df.empty: raise ValueError`,异常由 `server.py:116-117` 统一包成 `Error calling {name}: {e}` 返回 Agent),针对债券/转债补充:

| 场景 | 处理 |
|---|---|
| `bond_code` 子串误命中 | `issue_info` 精确过滤后为空 → `raise ValueError("未找到债券代码: {code}")` |
| `bond_info_detail_cm` bug | 不调用该接口 |
| 转债代码格式无效 | `_normalize_cb_code` → `raise ValueError` |
| 城投/非上市发行人解析正股失败 | 不报错,返回引导性提示 CSV |
| 日期参数格式错 | 复用 stock.py 的 `datetime.strptime` 校验 → `raise ValueError` |
| akshare 偶发限流/断连 | 不加重试(YAGNI,与 yahoo.py 一致),直接 raise 让上层重试 |
| 空数据 | `if df.empty: raise ValueError(...)` |
